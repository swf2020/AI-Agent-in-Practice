# AI Agent 社招面试题采集 Skill

## 概述

本 skill 从中文互联网平台系统采集 AI Agent / 大模型工程师社招面试真题，进行清洗去重、模块分类，最终生成一份可直接用于复习备考的 Markdown 题集。核心价值在于"真实性"——只保留有明确来源的实际被问题目，过滤掉泛泛而谈的内容。

## Step 2：清洗与去重

原始抓取结果往往参差不齐，这一步的目标是**只保留高质量的真实面试题**。

**保留标准（满足任意一条）：**
- 以"？"结尾的问句
- 包含关键词：如何 / 为什么 / 请设计 / 请解释 / 有什么区别 / 怎么实现 / 原理是

**过滤掉：**
- 泛泛而谈的问题，如"你了解大模型吗"、"谈谈你对 AI 的看法"
- 面试流程介绍、HR 问题（薪资期望、职业规划）
- 付费墙后的内容（不要尝试绕过）

**去重：**
- 语义相似度 >85% 的题目合并为一条
- 保留表达最清晰、最完整的版本
- 若同一题来自多个平台，保留一条并在来源中列出所有平台

**每道题格式：**
```
[来源:平台名][YYYY-MM] 题目内容？
```

**本次最多采集 200 题**，优先保留有明确答案思路或技术深度的题目。

## Step 3：分类映射

将清洗后的题目归入以下 8 个模块，选择**最贴近题目技术点**的一个模块：

| 模块 | 涵盖主题 |
|------|---------|
| Module 1 – Prompt / LLM 原理类 | Token、Temperature、微调、LoRA/QLoRA、Transformer、KV Cache、量化 |
| Module 2 – RAG 设计与优化类 | 向量检索、Embedding、混合检索、RAGAS、分块策略、重排序 |
| Module 3 – 工具调用类 | Function Calling、MCP、工具路由、tool_choice 策略 |
| Module 4 – Agent 架构设计类 | ReAct、记忆系统（短期/长期）、LangGraph、任务规划 |
| Module 5 – 多 Agent 设计类 | Multi-Agent、AutoGen、CrewAI、协作协议、冲突解决 |
| Module 6 – 工程落地类 | 流式输出、成本控制、可观测性、限流、安全防护、幻觉缓解 |
| Module 7 – 云端部署类 | Docker、Lambda/Serverless、CI/CD、K8s、API 网关 |
| Module 8 – 技术演进与视野类 | 历史演变、前沿趋势、开放性设计题、行业对比 |

无法明确归类的题目放入**"其他 / 综合类"**。

## Step 4：增量更新逻辑

在生成输出文件之前，先处理增量：

1. 检查输出目录是否已存在历史题集文件（`ai_agent_interview_questions_*.md`）
2. **若存在历史文件：**
   - 读取历史文件中已有的题目列表
   - 对比本次采集结果，识别出**新增题目**（历史文件中没有的）
   - 最终文件 = 历史题目 + 新增题目（去重合并）
   - 在文件头部注明"本次新增 N 题，来自 YYYY-MM-DD 之后的新内容"
3. **若不存在历史文件：**
   - 直接生成完整题集，注明"初次采集"

## Step 6：生成解答（按模块分 agent 并行）

题集文件生成后，**为每个有题目的模块启动一个独立的 sub-agent**，并行生成该模块下所有题目的解答思路与参考答案。这样做的目的：
- 不同模块的技术栈差异大，分 agent 可以让每个子任务专注于特定领域，答案质量更高
- 并行执行可以大幅缩短整体耗时

**并行启动规则：**
1. 遍历 8 个模块 + "其他/综合类"，只对**有题目**的模块启动 sub-agent
2. 每个 sub-agent 的任务描述如下（以 Module 1 为例，其他模块替换模块名称和题目列表即可）：

```
你是一个 AI Agent / 大模型技术专家。请为以下面试题提供详细的解答思路与参考答案。

模块：Module 1 - Prompt / LLM 原理类

题目列表：
- [来源:牛客][2025-04] Transformer 的 KV Cache 是什么？为什么能加速推理？
- [来源:小红书][2025-03] LoRA 的 rank 参数怎么选？rank=8 和 rank=64 的区别是什么？
...（列出该模块所有题目）

要求：
1. 每题包含三个部分：
   - **考察点**：一句话概括面试官想考察的核心能力
   - **解答思路**：2-3 步的回答框架，说明从什么角度切入
   - **参考答案**：300-500 字的详细回答，包含关键术语、代码示例（如适用）、对比表格（如适用）
2. 答案要体现生产级经验，不要只给教科书式定义
3. 如果题目涉及对比（如 A vs B），给出明确的差异总结表格
4. 标注"加分项"——能体现候选人深度的额外知识点

请将解答保存到文件：{output_base_dir}/answers/module_01_prompt_llm.md
```

**模块文件命名：**

## 注意事项

- 搜索失败不等于 skill 失败——跳过并继续，最后汇报哪些平台访问失败
- 不采集、不尝试绕过付费内容
- 题目质量 > 题目数量：200 题上限是为了保证精选
- 输出文件名含日期，方便按版本追溯历史
